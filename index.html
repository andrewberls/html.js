HTML.js

HTML.js is a tiny library for creating HTML fragments in JavaScript.

Tired of seeing code like this?

html = "<div class = '"+ divClass +"'><p>"+ content +"</p></div>"

So are we. That's why we created HTML.js. Instead of all that, why not write something like this?

div({ 'class': divClass }, function() {
    return p(content)
})

HTML.js is designed to be flexible. You can pass strings to it:

div("Hello World")

// => <div>Hello World</div>

You can specify ID and class with familiar CSS syntax:

div("#myId.myClass", "Hello World")

// => <div id='myId' class='myClass'>Hello World</div>

You can nest content:

ul(".myList", function() {
  return li("item one") + li("item two")
})

/*
  <ul class='myList'>
    <li>item one</li>
    <li>item two</li>
  </ul>
*/

You can add data attributes:

div({ id: 'myId', data: { name: 'Bart', city: 'Springfield' } }, "Hello World")

// => <div id='myId' data-name='Bart' data-city='Springfield'>Hello World</div>

Methods

HTML.js turns HTML tags into methods. You can call div(), a(), h3(), or just about every tag there is. Methods are defined on the window object, although HTML.js methods will not override existing methods. All methods are also available under the htmljs namespace, e.g., htmljs.div() and return strings.

Input Helpers

HTML.js comes with some utilities for input fields.

text_input({ name: 'age', placeholder: 'Enter your age' })
// => <input name='age' placeholder='Enter your age' type='text' />

checkbox_input({ name: 'vehicle', value: 'bike' })
// => <input name='vehicle' value='bike' type='checkbox' />

HTML.js defines <type>_input methods for the different types of HTML inputs such as text, checkbox, radio, etc.

CoffeeScript

HTML.js is even better if you're using CoffeeScript. This gives you the flexibility to write things like:

div 'data-name': 'John', ->
  h2("Hello World")

ul "#myList", ->
  li("item one") +
  li("item two")

Installation

Download html.min.js and link to it on your site!

<script src='path/to/html.min.js'></script>

Why HTML.js?

HTML.js is a small weekend project, but there is a chance it may be just what you need.

Keep things light

HTML.js is under 2K and does not depend on jQuery or any other library, although it works great with them!

Why not template library X / MV* framework Y?

There are a lot of libraries out there that do their job well. HTML.js is meant for constructing small snippets where adding in a framework might be overkill - it's not meant to render large templates or pages.
